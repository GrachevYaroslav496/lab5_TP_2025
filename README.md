# lab5_TP_2025

В этой лабораторной работе нужно смоделировать Scrum-процесс с использованием GitHub Issues.

## Цель работы - понять, как:

- формулировать требования в виде задач,
- разбивать работу на инкременты,
- планировать спринт,
- отслеживать прогресс и завершённость.

## Что нужно сделать
- Создать репозиторий для лабораторной работы
- Настроить работу через GitHub Issues
- Смоделировать один спринт по Scrum
- Описать результат в README.md


## Sprint Review

В ходе первого спринта были успешно завершены следующие задачи:
- Реализована главная страница приложения (Issue #1)
- Добавлен функционал добавления новых задач (Issue #2)

По итогу спринта пользователь может открыть приложение, увидеть интерфейс и добавлять задачи.  
Не успели завершить:
- Сохранение задач в localStorage (Issue #4)
- Отметка "выполнено" с сохранением состояния (Issue #5)

## Sprint Retrospective

**Что прошло хорошо:**  
- Чётко сформулированные задачи позволили быстро начать разработку.  
- Использование GitHub Issues помогло визуализировать прогресс.

**Что пошло не так:**  
- Недооценил сложность работы с localStorage — потратил больше времени, чем ожидал.  
- Не хватило времени на тестирование всех сценариев.

**Что улучшить в следующем спринте:**  
- Более реалистично оценивать story points.  
- Выделять отдельную задачу на тестирование.  
- Начинать с самых рискованных (технически сложных) задач в начале спринта.
